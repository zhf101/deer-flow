# PRODUCT.md

本文件记录仓库当前产品理解。涉及业务规则、用户流程、交互文案或造数语义时先阅读本文件。

## 产品定位

DeerFlow 是一个 Agent 工作区和运行平台。用户可以在工作区中与默认 Lead Agent 或自定义 Agent 对话，上传文件，调用工具，生成产物，管理技能、记忆、模型和 MCP 扩展。

GDP/datagen 是 DeerFlow 应用层中的造数业务域，用于把测试造数流程配置成可复用的场景，并逐步引入 Agent 辅助生成、校验、补全和执行。

## 核心用户流程

### 通用 Agent 工作区

1. 用户进入 Workspace。
2. 新建或选择线程。
3. 选择模型和运行模式。
4. 发送消息或上传文件。
5. Gateway 创建 run，Lead Agent 调用模型、工具、子代理和沙箱。
6. 前端通过 SSE 展示消息、工具进度、产物和 token 用量。
7. 运行结束后可查看产物、反馈、继续对话或管理记忆。

### GDP/datagen 配置流程

1. 配置基础信息：系统、环境、服务端点、数据源。
2. 配置 HTTP 源或 SQL 源，并测试可用性。
3. 创建造数场景，定义输入参数和编排步骤。
4. 步骤可以调用 HTTP、SQL、断言、变量转换等能力。
5. 配置结果映射和批量配置。
6. 保存、发布、运行场景，查看运行历史和失败原因。

### GDP/datagen Agent 流程

1. 用户描述造数目标或提供已有接口、SQL、场景线索。
2. Agent 检索场景目录、数据源、基础设施和业务记忆。
3. Agent 生成或补全场景草稿。
4. 用户确认后发布或继续调整。
5. Agent 可在不同阶段使用受限工具和 MCP 能力辅助分析。

## 业务对象

| 对象 | 含义 |
|---|---|
| 系统 | 业务系统编码和名称，是环境、端点、数据源的上层归属 |
| 环境 | 如测试环境、预发环境，不同环境有不同服务端点和数据源 |
| 服务端点 | HTTP 调用的基础地址、认证和连接信息 |
| 数据源 | SQL 执行所需的数据库连接配置 |
| HTTP 源 | 可复用的 HTTP 请求模板和响应映射 |
| SQL 源 | 可复用的 SQL 模板、参数、解析结果和安全策略 |
| 场景 | 造数的发布和执行单位 |
| 步骤 | 场景内部的串行执行单元 |
| 任务 | 对多个造数动作或子任务的编排 |
| Agent 记忆 | 业务规则、字段含义、历史经验等可复用事实 |

## datagen 业务规则

- 环境是多数造数场景的基础输入，用于解析服务端点和数据源。
- 场景执行应返回人类可理解的成功结果或失败原因。
- SQL 执行必须经过安全策略和参数处理，不允许绕开运行时执行器直接拼接危险 SQL。
- HTTP 和 SQL 步骤的输出应进入运行上下文，后续步骤通过变量引用消费。
- 断言失败应中止当前场景，并保留足够上下文便于定位。
- 批量造数按行独立执行，每行独立保留输入、结果和失败原因。
- Agent 生成的配置必须经过校验和用户确认，不应直接静默发布。

## 开发期产品策略

- 当前阶段优先保证模型、字段和流程简洁一致。
- 不为旧字段和旧 API 做兼容包袱。
- 用户可接受开发数据清空重建。
- 变更业务字段时必须同步前端显示、表单、校验、后端模型、测试和文档。
