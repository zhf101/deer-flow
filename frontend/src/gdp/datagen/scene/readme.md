### scene-dashboard.tsx
/**
 * ============================================================================
 * 场景编排 - 场景管理仪表盘（列表首页）
 * ============================================================================
 *
 * 【页面定位】场景编排模块的入口页面，展示所有已创建的场景列表。
 * 【是否新增】新增页面，场景编排功能专属。
 *
 * UI 内容：
 *   1. 顶部标题栏 —— "造数编排" 标题 + "生成演示配置" 按钮 + "新增场景" 按钮
 *   2. 搜索/筛选栏 —— 关键词搜索输入框 + 状态下拉筛选（草稿/已发布/已停用）+ 刷新按钮
 *   3. 场景列表表格 —— 列：场景名称、业务分类、状态、当前版本、最后更新、操作
 *   4. 操作按钮 —— 查看详情、编辑、更多（复制场景、删除场景）
 *   5. 分页控件 —— 上一页/下一页
 *   6. 复制场景对话框 —— 输入新场景编码后复制
 *   7. 删除确认对话框 —— 二次确认删除
 *   8. 生成演示场景 —— 点击后自动创建一个电商下单核心链路的演示场景
 *
 * 交互：
 *   - 点击行 → 进入只读查看模式
 *   - 点击编辑 → 进入场景编辑器（SceneEditor）
 *   - 点击新增 → 创建空白场景并进入编辑器
 *
 * 数据来源：listScenes / createScene / copyScene / deleteScene API
 */

### scene-editor.tsx 
/**
 * ============================================================================
 * 场景编排 - 场景编辑器（主容器页面）
 * ============================================================================
 *
 * 【页面定位】场景编排的核心编辑页面，作为所有配置步骤的容器和状态管理中心。
 * 【是否新增】新增页面，场景编排功能专属。
 *
 * UI 结构（整体布局）：
 *   ┌──────────────────────────────────────────────────┐
 *   │  SceneEditorSidebar（左侧步骤导航栏）              │  │  SceneEditorHeader（顶部标题栏）   │
 *   │                                                  │  │──────────────────────────────────│
 *   │  步骤 0: 基础配置 → BasicInfoPanel                │  │  主内容区（按 currentStep 切换）     │
 *   │  步骤 1: 参数配置 → InputSchemaPanel              │  │                                  │
 *   │  步骤 2: 逻辑编排 → LogicOrchestrationStep        │  │  当前步骤对应的面板组件            │
 *   │  步骤 3: 结果输出 → ResultMappingPanel            │  │                                  │
 *   │  步骤 4: 批量设置 → BatchConfigPanel              │  │                                  │
 *   │                                                  │  │                                  │
 *   │  [保存] [发布] [执行]                              │  │                                  │
 *   └──────────────────────────────────────────────────┘
 *
 * 核心职责：
 *   1. 状态管理 —— 维护 scene 全局状态（SceneDefinition），包括步骤列表、参数结构、结果映射等
 *   2. 数据加载 —— 通过 sceneCode 从后端加载已有场景（getScene），或创建默认场景
 *   3. 自动保存 —— 步骤切换时自动保存（save），支持手动保存和发布（publishScene）
 *   4. 步骤导航 —— 控制 currentStep 在 5 个配置步骤间切换，切换前自动校验并保存
 *   5. 资源预加载 —— 加载 SQL 模板列表、HTTP 接口列表、SQL 接口列表供子组件使用
 *   6. 校验提示 —— 实时调用 validateSceneDraft 校验场景配置，发布时检查 ERROR 级别问题
 *   7. 执行入口 —— 已发布场景显示执行按钮，打开 SceneRunDialog
 *   8. 数据归一化 —— normalizeScene 确保从后端加载的数据结构完整（填充默认值）
 *
 * Props：
 *   - sceneCode: 场景编码，有值表示编辑已有场景，null 表示新建
 *   - onBack: 返回列表页回调
 *   - readOnly: 只读查看模式，禁用所有编辑和保存操作
 *
 * 子组件依赖：
 *   - SceneEditorSidebar / SceneEditorHeader（布局框架）
 *   - BasicInfoPanel / InputSchemaPanel / LogicOrchestrationStep / ResultMappingPanel / BatchConfigPanel（5 个步骤面板）
 *   - SceneRunDialog（执行场景对话框）
 */

### input-schema-panel.tsx
/**
 * ============================================================================
 * 场景编排 - 步骤 1：参数配置面板
 * ============================================================================
 *
 * 【页面定位】场景编辑器第 2 步（currentStep=1），定义场景的输入参数结构（inputSchema）。
 * 【是否新增】新增组件，场景编排功能专属。
 *
 * UI 内容：
 *   1. 顶部工具栏 —— "参数结构" 标题 + "贴入 JSON 生成" 按钮 + "新增参数" 按钮
 *   2. 参数树形编辑器 —— 递归渲染 InputFieldTreeItem，支持嵌套对象和数组：
 *      每个字段行：
 *      ┌─────────────────────────────────────────────────┐
 *      │ [参数编码] [中文描述] [类型下拉] [必填开关] [删除] [+] │
 *      └─────────────────────────────────────────────────┘
 *      - 类型下拉：string / number / boolean / date / object / array
 *      - object/array 类型可添加子参数（递归嵌套，带缩进左边框线）

 *
 * 内部子组件/函数：
 *   - InputFieldTreeItem：递归渲染的参数树节点，支持增删改和嵌套
 *   - jsonToFields：将 JSON 对象递归转换为 InputFieldDefinition 数组
 *   - parseJsonWithComments：解析带 // 注释的 JSON，提取注释作为中文描述
 *
 * 被谁使用：SceneEditor 在 currentStep=1 时渲染
 */

/**

 * 场景编排 - 逻辑编排：步骤详细配置面板
 * ============================================================================
 *

 *
