# 智能造数平台功能拆分文档 V1

## 1. 文档目标

本文档用于将智能造数平台 V1 需要实现的功能做结构化拆分，形成可管理、可排期、可认领的功能清单。

本文档回答的是：

- V1 到底要实现哪些功能
- 这些功能分别属于哪一层
- 哪些是主链路功能
- 哪些是配套治理功能

## 2. 功能拆分原则

### 2.1 以主链路为中心

V1 功能拆分围绕平台主链路：

```text
聊天探索
  ↓
FlowDraft
  ↓
预检
  ↓
受控试跑
  ↓
模板草稿
  ↓
审核发布
  ↓
标准执行
```

### 2.2 按层次拆分

功能按五个层次组织：

- 探索前台
- 业务内核
- 执行引擎
- 治理与权限
- 管理与查看

## 3. 探索前台功能

### F1. 聊天探索入口

需要实现：

- 聊天页入口
- 多轮澄清需求
- 结构化澄清表单
- 聊天历史承载

### F2. FlowDraft 生成

需要实现：

- 从聊天整体生成初版 `FlowDraft`
- 支持生成业务意图图
- 支持生成技术实现图
- 支持待确认项列表

### F3. FlowDraft 交互修正

需要实现：

- 用户通过聊天修正 FlowDraft
- 用户通过节点详情修正 FlowDraft
- 系统按问题类型推荐修正路径

### F4. FlowDraft 版本快照与 diff

需要实现：

- 初版快照
- 重收敛快照
- 试跑前快照
- 试跑成功后快照
- 业务意图层 diff
- 技术实现层 diff

### F5. 五层前端视图

需要实现：

- `FlowDraftGraph`
- `FlowStepDesign`
- `PreflightCheckView`
- `TrialExecutionDetail`
- `ToolTraceView`

## 4. 业务内核功能

### F6. FlowDraft 中心

需要实现：

- `FlowDraft` 模型
- `business_graph`
- `technical_graph`
- 待确认项持久化
- 预检摘要持久化

### F7. 技术节点模型

需要实现：

- `http_step`
- `sql_step`
- `confirm`
- `mapping`
- `start`
- `end`

每个技术节点至少承载：

- `design_intent`
- `resolution_rationale`
- `resolved_execution_plan`
- `editable_fields`

### F8. 模板中心

需要实现：

- `Template`
- `TemplateRevision`
- 模板草稿生成
- 模板版本流
- 发布版本管理

### F9. Run 中心

需要实现：

- `Run`
- `RunStep`
- 运行状态
- 输入输出快照
- 运行详情聚合

### F10. Snapshot / Diff 中心

需要实现：

- `FlowDraftSnapshot`
- `BusinessGraphDiff`
- `TechnicalGraphDiff`

## 5. 执行引擎功能

### F11. HTTP Resolver

需要实现：

- 从 `design_intent` 收敛 HTTP 执行方案
- 参数模板收敛
- 默认值补全
- 提取规则生成
- `resolution_rationale`
- `blocking_issues`

### F12. HTTP Executor

需要实现：

- 按 `resolved_execution_plan` 执行 HTTP 调用
- 请求快照
- 响应快照
- 结果提取
- 指标统计

### F13. SQL Resolver

需要实现：

- 从 `design_intent` 收敛 SQL 执行方案
- schema 理解
- 历史 SQL 参考
- lane 判定
- 风险判定
- 确认要求判定
- `resolution_rationale`

### F14. SQL Executor

需要实现：

- 按 `resolved_execution_plan` 执行 SQL
- SQL 快照
- 参数绑定快照
- 查询/变更结果快照
- 结果提取
- 审计载荷输出

### F15. 控制节点执行

需要实现：

- `confirm` 节点
- `mapping` 节点
- `start` 节点
- `end` 节点

## 6. 治理与权限功能

### F16. 角色与 `systemShort`

需要实现：

- `system_admin`
- `domain_admin`
- `user`
- `admin_system_scopes`

### F17. 对象级鉴权

需要实现：

- 基于 `systemShort` 的对象过滤
- 资产 / 模板 / Run 可见范围控制
- 审核路由控制

### F18. SQL 资产审核后生效

需要实现：

- SQL 资产逻辑对象
- SQL 资产版本对象
- 提交审核
- 审核通过切换生效版本
- 审核驳回

### F19. 试跑前预检

需要实现：

- `route_pending`
- `asset_pending`
- `param_pending`
- `governance_blocked`
- `dependency_incomplete`
- `mapping_incomplete`

### F20. 危险 SQL 确认

需要实现：

- mutation 前确认
- query 风险校验
- 用户确认记录

### F21. 审计中心

需要实现：

- SQL 审计记录
- 审计列表
- 审计详情
- 管理员可见性控制

## 7. 管理与查看功能

### F22. HTTP 资产中心

需要实现：

- HTTP 资产列表
- HTTP 资产详情
- 创建、更新、删除
- 测试调用

### F23. SQL 资产中心

需要实现：

- SQL 资产列表
- SQL 资产版本列表
- 创建、修改、复制
- 测试连接 / 测试 SQL
- 审核状态展示

### F24. 模板管理台

需要实现：

- 模板列表
- 草稿、待审核、已发布、历史版本视图
- 模板详情
- 模板版本详情
- 审核动作

### F25. 模板执行入口

需要实现：

- 已发布模板选择
- `input_schema` 对应表单渲染
- 创建 Run

### F26. Run 详情页

需要实现：

- Run 状态
- RunStep 列表
- 每步输入输出
- SQL 审计摘要

### F27. 关系视图

需要实现：

- 模板详情页中的资产引用提示
- 资产详情页中的模板反向引用提示

## 8. 平台级基础功能

### F28. RunRuntimeBridge

需要实现：

- `task_id <-> run_id` 映射
- 从探索态 runtime 投影到执行态 Run
- 聚合 trace 到 Run 视图

### F29. Workspace 隔离

需要实现：

- `TaskWorkspace`
- `RunWorkspace`
- 探索态与执行态文件隔离
- 产物与 `run_id` 关联

### F30. 事务与版本锁定

需要实现：

- 模板草稿生成事务
- SQL 资产审核切换事务
- Run 初始化事务
- 资产版本锁定与执行快照

### F31. 前端状态模型拆分

需要实现：

- 探索态 store
- 执行态 store
- 运行页与聊天页状态解耦

## 9. V1 最小闭环

如果只看 V1 最小闭环，至少必须打通：

1. 聊天探索
2. FlowDraft 生成
3. 待确认项与预检
4. HTTP / SQL 收敛与试跑
5. Run 创建与详情
6. 模板草稿自动生成
7. 模板审核发布
8. 已发布模板执行
9. SQL 审计

## 10. 功能总结

一句话总结：

V1 功能不是若干孤立页面，而是一条从探索态到执行态、从试跑到发布、从资产到审计的完整平台链路。所有功能拆分都必须围绕这条闭环服务。
