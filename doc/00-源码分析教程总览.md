## 主线理解

DeerFlow 不只是一个聊天应用，而是一个长时序 Agent 运行时。它的核心链路可以概括为：

```text
前端请求
  -> Gateway API
  -> start_run / run_agent
  -> make_lead_agent
  -> LangChain create_agent / LangGraph 状态图
  -> ThreadState + middleware chain
  -> tools / sandbox / MCP / subagents / skills / memory
  -> checkpointer + stream bridge + event store
  -> SSE 流式返回给前端
```

学习源码时不要只看“某个类做什么”，更要看它解决的工程问题：

- `ThreadState` 解决跨轮次状态、并发合并、流式替换、恢复持久化。
- 工厂函数解决每请求动态配置、热重载、无共享可变状态。
- 中间件链把复杂 Agent 能力拆成可测试、可插拔的横切关注点。
- 沙箱用目录隔离和虚拟路径翻译控制文件/命令执行风险。
- 子 Agent 用隔离上下文和后台执行解决长任务的并行和上下文膨胀。
- 记忆系统异步、防抖、带置信度地提取跨会话事实。
- MCP 与 Skills 分别解决“工具扩展”和“方法论注入”。
- Gateway API 是前端、鉴权、文件、配置管理和 LangGraph 运行时的系统边界。
