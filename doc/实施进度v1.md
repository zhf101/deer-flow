## 3. 已完成内容

### 3.1 需求与方案收敛

已完成：

- 明确平台定位：智能造数平台不是通用 Agent 系统，而是“聊天探索 -> 业务 DAG -> 试跑 -> 模板沉淀 -> 审核发布 -> 标准执行”的闭环平台
- 明确 `Xagent` 负责探索前台，`datamakepool` 负责平台业务内核
- 明确主 DAG 不是工具调用 DAG，而是业务 DAG
- 明确两层图：
  - `business_graph`
  - `technical_graph`
- 明确五层视图：
  - `FlowDraftGraph`
  - `FlowStepDesign`
  - `PreflightCheckView`
  - `TrialExecutionDetail`
  - `ToolTraceView`

### 3.2 权限与审核设计

已完成：

- 明确保留三类角色：
  - `system_admin`
  - `domain_admin`
  - `user`
- 明确保留 `systemShort` 作为对象归属与审核路由字段
- 明确保留 SQL 资产审核后生效
- 明确保留模板发布作为正式安全门
- 明确保留 Run 查看范围限制
- 明确保留审计仅系统管理员可见

### 3.3 运行时与执行态设计

已完成：

- 明确 `Task` 是探索态宿主
- 明确 `Run` 是执行态宿主
- 明确 V1 采用“业务上双宿主、基础设施上分层宿主”
- 明确 `DAGExecution` 在 V1 只保留探索态用途
- 明确需要 `RunRuntimeBridge`
- 明确需要 `TaskWorkspace / RunWorkspace`

### 3.4 执行器与技术节点设计

已完成：

- 明确 V1 技术节点类型：
  - `http_step`
  - `sql_step`
  - `confirm / mapping / start / end`
- 明确每个技术节点都包含：
  - `design_intent`
  - `resolution_rationale`
  - `resolved_execution_plan`
- 明确 `HTTP Resolver / Executor`
- 明确 `SQL Resolver / Executor`
- 明确试跑成功后必须固化执行方案
